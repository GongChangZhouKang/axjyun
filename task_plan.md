# 采购管理单号统一

## 目标
将采购需求、采购计划、采购订单统一为“拼音简写 + yyyyMMdd + 四位流水”，并保证原型中的示例数据、生成逻辑和关联链路一致。

## 阶段
- [x] 阶段 1：确认需求与设计方案
- [x] 阶段 2：盘点全部采购单号及生成逻辑
- [x] 阶段 3：实现统一编号规则并迁移示例数据
- [x] 阶段 4：全量检索、构建与页面验收

## 已确认规则
- 采购需求：`XQyyyyMMdd####`
- 采购计划：`JHyyyyMMdd####`
- 采购订单：`DDyyyyMMdd####`
- 各单据类型按自然日独立流水，从 `0001` 开始。
- 页面结构与视觉方向不变。

## 错误记录
| 错误 | 尝试 | 处理 |
|---|---:|---|
| 误读不存在的 `shared.ts` | 1 | 已定位实际文件为 `shared.tsx`，不影响实现 |
| 无输出路径的 esbuild 无法处理 CSS | 1 | 改用项目 TypeScript 配置进行静态检查，再用应用就绪检查验证 |
| 全仓 TypeScript 检查缺少 React 类型声明 | 1 | 属于项目环境既有问题；改用当前原型入口的独立打包校验 |
| 应用就绪脚本在 Windows 启动服务时 `spawn EINVAL` | 1 | 改为直接启动 Vite 并在内置浏览器验收 |
| 浏览器只读脚本不可使用 `HTMLInputElement instanceof` | 1 | 改用 DOM 属性读取，后续验收通过 |
