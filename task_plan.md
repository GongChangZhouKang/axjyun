# 采购订单一对多生成入库单

## 目标
在采购订单页新增独立生成入库单页面，支持一次生成多张入库单与按剩余数量后续追加；累计生成数量不得超过采购订单数量，并同步采购订单列表、订单详情、入库管理和链路追踪。

## 已确认需求与设计决策
- 产品需求已由用户确认：采购订单可一对多生成入库单；首张草稿默认带入全部剩余数量；新增草稿优先带入未分配余量，无余量以 0 数量创建并提示重新分配。
- 范围：前端原型内完成数据、页面、校验、状态汇总、链路追踪和归档；不新增真实后端、审批或入库确认状态流转。
- 数量规则：非负整数；阻止空草稿、负数、非整数和累计超量；零数量明细不生成。
- 关系事实源：以 FlowRow.from + InboundTrace 表达订单—入库单关系；订单来源分配不再承担一对一入库关联。
- 样式决策：本次新增和触达区域先参考当前采购管理页面；DESIGN.md 暂不作为严格视觉约束，后续由用户统一调整。
