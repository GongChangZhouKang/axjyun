# Findings

## 输入证据
- 截图为 1920×900 的中文政企后台「合同详情」页面，视觉骨架由 64px 顶栏、约 200px 侧栏、32px 标签栏和高密度内容区组成。
- 原始附件实际像素为 1912×901；上行的 1920×900 为肉眼近似，后续以附件像素和样式采集值为准。
- 截图中的内容容器左边缘约为 x=212，顶部业务摘要起点约为 y=122；摘要卡高约 96px，卡间距约 6px，验证了“低圆角、浅灰填充、紧凑横向信息块”的主题结构。
- 选中页签使用蓝色文字与约 4px 下划线，主按钮约 62×34px；普通输入框与按钮高度约 32–34px，和采集数据中的 14px 字号、4px 圆角一致。
- 视觉主调为白色、冷灰与亮蓝；强调色同时承担链接、选中态、主按钮和焦点态。
- 样式采集数据显示主文本 `rgb(81, 90, 110)`（`#515A6E`），品牌/链接蓝 `rgb(45, 140, 240)`（`#2D8CF0`），主按钮蓝 `rgb(18, 155, 255)`（`#129BFF`）。
- 常用背景为 `#FFFFFF`、`#F7F7F7`、`#F4F5F7`、`#F3F3F3`、`#F9F9F9`；常用边框为 `#DCDEE2` 与 `#C5C8CE`。
- 正文字体事实源为 `Helvetica Neue, Helvetica, PingFang SC, Hiragino Sans GB, Microsoft YaHei, Arial, sans-serif`；主体 14px/21px/400，表头常用 14px/21px/700，次级标题可用 16px/24px/400。
- 高频圆角为 4px 和 3px；16px 只出现在少量特殊容器，不能升级为常规组件圆角。
- 高频间距集中在 4/6/7/8/10/12/14/16/20/24px；主题将用 2px 基础刻度并保留 7px 密度微调 token。
- 常用阴影为 `0 1px 6px rgba(0,0,0,.2)` 和 `0 4px 12px rgba(0,0,0,.15)`，应仅用于浮层；常规卡片主要依赖填充色和边框分层。
- 常规过渡为 200ms，较明显的进入动效为 400ms `cubic-bezier(.4,0,.2,1)`；加载旋转 1s linear。
- 像素统计再次验证高频色：`#FFFFFF`、`#F2F2F2`、`#F5F7F9`、`#F5F7FB`、`#F4F5F7`、`#DCDEE2`、`#515A6E`、`#129BFF`、`#2D8CF0`，并从状态标签采样到 `#6CE21F`。

## 系统级截图分析（第一组）
- 合同列表确认全局“数据密集列表”模式：顶部二级页签约 44px，高亮为蓝字 + 4px 下划线；筛选容器使用 `surface-soft`，多列 34px 控件，右侧放查询/重置/更多；表格使用浅灰表头和约 48px 斑马纹数据行。
- 合同列表补充状态与操作语义：执行中为亮绿标签，未执行/终止为灰标签；默认操作链接为蓝色，危险或终止操作为红色；同一行允许文字链接组合但不使用实心按钮。
- 添加用户弹窗确认模态系统：页面遮罩约 45%–50% 黑灰；大型模态宽约 900px、高约 710px，16px 圆角，白色表面，顶部标题栏约 52px，底部固定操作栏约 58px。
- 弹窗内部为 44px 页签 + 可滚动内容区；表单采用两列标签/控件布局，标签右对齐，必填星号为红色，输入/选择器 34px；角色区使用边框分组和密集复选框矩阵。
- 紫色圆形拍照按钮属于“人脸照片录入”流程专用强调，不应升级为全局第二品牌色；应记录为 feature-specific token。

## 系统级截图分析（第二组）
- 在职员工页确认左侧“树 + 右侧列表”的主从布局：树面板约 300px，右侧自适应；树节点选中为浅蓝底，层级箭头、机构圆形图标与竖向滚动条均属于导航组件族。
- 批量操作区确认工具栏可容纳多个主色按钮，但这些是同一批量任务簇；按钮间距约 12px，高 34px，包含下拉分裂按钮。设计规则应限制为“批量工具栏例外”，避免普通页面滥用多个主按钮。
- 员工表格补充橙色警告/待处理语义（待签署）、红色危险操作（离职）和蓝色普通操作（编辑）；主岗前的小旗/机构图标属于业务标识，不应进入通用语义色。
- 员工入职页进一步确认斑马纹行：白色与约 `#F4F5F7`/`#F5F5F6` 交替；成功状态采用明亮绿色文本而非胶囊，说明同一语义允许“标签”与“纯文本”两种密度变体。
- 分页固定在表格右下方；包含总数、前后翻页、页码、省略号与每页条数选择，控件同样使用 34px/4px 基础规则。

## 系统级截图分析（第三组与员工样式数据）
- 装备出库页确认“编辑表单”模式：内容区左对齐、字段最大宽度受控；分组标题使用 5px 左侧蓝色竖条 + 16px/24px 粗体；文本域高约 52px；明细表空状态占据剩余工作区。
- 页面级提交操作固定在底部白色操作栏，含取消/暂存/确认；主操作仍为蓝色，次操作为白底描边。该固定栏与弹窗底栏属于同一 action-footer 组件族。
- 员工样式采集再次确认全局基础：正文 `#515A6E`、白色、`#F4F5F7`、`#F8F8F9`、`#129BFF`、边框 `#DCDEE2/#C5C8CE`、3/4/16px 圆角、1px 边框、200ms 过渡、两级浮层阴影。
- 新增精确表面 token：斑马纹 `#F8F8F9`、树节点/侧栏选中 `#F0FAFF`；新增浮层 token：遮罩 `rgba(55,55,55,.60)`、modal 阴影 `0 0 12px rgba(0,0,0,.16)`。
- 样式数据给出拍照上传专用渐变：`#667EEA → #764BA2`；仅限人脸/照片录入等 feature-specific 浮动操作。
- 像素证据补充语义色：危险/终止使用 `#ED4014`、`#F52C2C` 或高强调 `#FF0000`；待签署橙色最接近 `#F59A23`；已入职文本为 `#23D170`；执行中标签为 `#6CE21F`。生产语义仍需为小字号校准对比度。
- 员工样式新增圆角 2px（小图片/微型元素）、6px（局部分组），但 4px 仍为全局控件与卡片默认，16px 仍只用于大型模态。

## 可访问性校准
- `#515A6E` / `#FFFFFF` 对比度 6.91:1，可作为默认正文。
- `#808695` / `#FFFFFF` 为 3.64:1、`#999999` / `#FFFFFF` 为 2.85:1、`#C5C8CE` / `#FFFFFF` 为 1.68:1，只能用于次要元数据、占位或禁用态，不能承载关键正文。
- 源截图白字 / `#129BFF` 仅 2.93:1，白字 / `#2D8CF0` 仅 3.43:1；为满足 Stitch 的 WCAG 校验，保留二者为 source-exact/非文本强调 token，推荐按钮与文字链接使用 `#096DD9`（白字对比度 5.00:1）。
- 源状态绿 `#6CE21F` 配白字仅 1.68:1；推荐改为深色字 `#1F2A10`（8.97:1），或浅绿底 `#F6FFED` + 深绿字 `#237804`（5.44:1）。

## 约束
- `DESIGN.md` 是主题事实源，其他文件必须从中派生。
- 主题目录需要包含 `theme.json`、`assets/tokens.json`、`style.css`、`tw.css`、`index.tsx` 和本地预览资源。
- 不引用主题目录外资产，不引入与主题无关的 UI 库。

## 项目结构
- 项目通过主题目录自动发现入口，不需要手工注册；标准参考为 `src/themes/linear/`。
- 标准入口将 `theme.json.display` 映射到 `DesignMdBatchShowcase`，并静态导入本地预览资源。
- 通用展示组件可呈现色板、字体、间距、圆角、阴影、边框、组件片段、Do/Don't，并自带预览图放大交互；新主题应复用该结构。

## Google Stitch / DESIGN.md 规范核对
- Google Labs 已在 2026-04 开源 DESIGN.md alpha 草案；官方定位是“机器可读 YAML tokens + 人类可读 Markdown 设计理由”。
- YAML token 是规范值，正文负责解释意图与使用边界；本主题会在正文为每类 token 提供“推荐 / 允许 / 禁止”三级规则。
- 规范顶层 token 包含 `colors`、`typography`、`rounded`、`spacing`、`components`；组件状态应拆成相关但独立的条目，如 `button-primary` 与 `button-primary-hover`。
- 正文规范顺序为 Overview → Colors → Typography → Layout → Elevation & Depth → Shapes → Components → Do's and Don'ts；额外章节允许保留但不能重复标题。
- 官方 CLI 可 lint DESIGN.md、校验 token 引用与组件对比度，并可直接导出 Tailwind v4 `@theme` CSS；Windows 应使用 `npx -p @google/design.md designmd ...` 避免 `.md` 文件关联冲突。
- 官方草案当前为 alpha，主题文档需标注版本并避免依赖未公开的扩展语法。
