# 安保管理平台系统级设计主题

## 目标
以用户提供的安保管理平台代表性截图与样式采集数据为视觉事实源，创建并持续升级符合项目主题规范与 Google Stitch 使用习惯的系统级主题资源，包括 `DESIGN.md`、Tailwind CSS v4 主题、结构化 tokens、主题元数据和可运行设计系统演示页。

## 已确认边界
- 目标用户：需要延续现有智慧安保后台视觉语言的产品、设计与前端人员。
- 核心任务：将截图与样式数据沉淀为可复用主题，并直观看到组件使用效果。
- 范围：主题文档、tokens、Tailwind v4、主题元数据、可运行演示页；不复刻完整合同业务页面。
- 内容来源：用户首张合同详情截图、样式采集 JSON、后续 5 张合同列表/员工列表/入职状态/添加用户弹窗/装备出库截图、项目主题规范与现有参考主题结构。
- 交互：演示页提供轻量组件状态切换，不扩展业务流程。
- 验收重点：token 与证据一致；推荐/允许/禁止规则齐全；主题页可打开、可读、可交互、无明显布局错误。

## 设计基底
- 用户明确指定截图为视觉基底：白色主画布、冷灰层级、亮蓝主色、紧凑 14px 中文后台排版、轻边框和低阴影。
- 不引入额外品牌风格，不使用渐变作为常规 CTA，不扩大圆角或卡片阴影。

## 错误记录
| 错误 | 尝试 | 处理 |
|---|---:|---|
| 依赖包提供的 `dependencies/bin/npx.cmd` 路径不存在 | 1 | 改为定位随 Node runtime 附带的 npm/npx 入口，或使用 `pnpm dlx` 调用官方 CLI |
| `check-app-ready.mjs` 在 Windows 启动 Vite 时返回 `spawn EINVAL` | 1 | 沿用项目已知兼容方案：直接启动 Vite，再用内置浏览器完成页面、交互与控制台验收 |
| 内置浏览器不支持 `networkidle` load state | 1 | 页面导航后直接使用 DOM snapshot 与目标元素状态进行确定性验收 |
| Token 明细页的截图捕获两次超时 | 2 | 未重复扩大截图；改用 DOM、computed style 与无溢出检查验收，组件实景桌面/移动截图此前已成功 |
| 标准 `pnpm run typecheck` 被依赖策略中的 `esbuild` ignored-builds 阻断 | 1 | 未修改项目依赖策略；改用已运行的 Vite 页面编译、浏览器控制台与主题专项校验确认本次实现 |
| 直接运行全仓 TypeScript 检查因项目缺少 `@types/react` 产生跨全部主题的既有错误 | 1 | 判定为仓库级依赖基线问题，不在本主题任务中扩改；本主题 Vite 热编译与交互验收通过 |
