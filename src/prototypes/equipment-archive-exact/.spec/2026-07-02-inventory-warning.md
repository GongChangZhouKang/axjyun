# 库存管理与库存预警决策快照

## 产品需求

- 目标用户：装备管理员。
- 核心任务：查询装备库存，并维护库存预警阈值。
- 页面范围：沿用现有平台框架，新增库存管理页面。
- 原有能力：仓库选择、装备名称/分类/状态/供应商筛选、查询、重置、导出、排序和分页。
- 新增能力：装备单独设置预警数量；勾选多件装备后批量统一覆盖预警数量。
- 预警规则：预警数量为 0 时不启用预警；预警数量大于 0 且 0＜库存数量≤预警数量时显示“库存不足”；库存为 0 时显示“无货”。
- 内容来源：用户截图、现有装备档案页面及原型标注资料。
- 验收重点：视觉与现有平台一致；单独设置和批量设置流程可完整操作；预警状态清晰。

## 设计决策

- 设计基底：当前原型 `DESIGN.md`。
- 信息架构：库存管理作为装备管理子菜单和顶部页签接入，并提供独立预览路径。
- 交互路径：列表行操作进入单件设置；勾选列表后使用工具栏进入批量设置。装备名称仅用于展示，不触发预警设置。
- 批量规则：输入一个非负整数预警数量，统一覆盖所有选中装备的已有设置。
- 视觉方向：严格继承平台的筛选区、斑马纹表格和蓝色操作色，仅用克制的红色强调库存不足。

## 实施边界

- 本轮实现库存页面、筛选、分页、预警展示、单件设置和批量设置。
- 本轮不实现消息推送、预警历史和自动补货。
