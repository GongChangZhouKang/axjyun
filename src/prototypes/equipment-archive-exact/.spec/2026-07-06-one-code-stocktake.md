# 一物一码连续录入与盘点核对

## 产品需求确认

- 目标用户：有盘点和物码维护权限的仓库管理员。
- 核心任务：使用扫码枪连续录入物码，并在盘点时判断扫描码与库内正式物码的关系。
- 范围：新增桌面端“添加一物一码”和“盘点一物一码核对”两个 Page，完成扫码模拟、即时入列、关系识别、暂存、回写和提交确认。
- 不包含：移动端、真实接口、扫码枪驱动、强制改绑和后端持久化。
- 内容来源：现有库存盘点任务、装备档案和原型内模拟物码台账。
- 验收重点：连续扫码焦点不中断；匹配、新增、冲突关系准确；重复码不重复计数；实盘数量正确回写。

## 设计方案确认

- 设计基底：当前原型 `DESIGN.md`，沿用蓝灰色 Element Plus 管理后台语言。
- Page 1：弹窗式独立页面，输入区、统计区、紧凑列表和底部确认操作；列表去掉“添加方式”列。
- Page 2：单列表展示关系，不使用左右两栏；关系单元格以绿色对勾、蓝色加号和红色警告分别表达库内已有、本次新增、冲突阻断。
- 高频交互：输入框默认聚焦，Enter 完成一次扫码，记录最新置顶，处理完成后恢复焦点。
- 盘点口径：实盘数量等于库内匹配数加本次新增数；冲突码不计入。

## 最终决策

- 未知码先暂存为待建档，提交整张盘点单前统一确认创建。
- 已存在但不属于当前装备、当前仓库或非在库状态的物码直接阻断，不提供强制匹配。
- 使用 Edge 完成交互与视觉验收。
