# 项目装备盘点｜产品需求确认快照

## 产品需求
- 目标用户：项目负责人、班长。
- 核心任务：在 APP 中完成项目装备和甲方装备盘点。
- 统一流程：新增盘点 → 录入 → 提交 → 查看结果。
- 项目装备来源：当前项目已领用的装备。
- 甲方装备来源：当前项目已登记的甲方装备；允许在盘点过程中补充登记。
- 项目装备盘点字段：装备名称、领用数量、盘点数量、差异数量。
- 甲方装备盘点字段：装备名称、登记数量、盘点数量、差异数量。
- 差异数量由系统自动计算：盘点数量减去领用数量或登记数量。
- 结果规则：差异为 0 标记正常，差异大于 0 标记盘盈，差异小于 0 标记盘亏。
- 提交后形成盘点结果，不另外建立异常处理流程。

## 实施边界
- 不设置“开始盘点”步骤。
- 不锁定仓库或项目装备业务。
- 不生成库存出入库单。
- 不包含独立的异常处理、复核或审批流程。

## 设计方案
- 设计基底：沿用当前原型 `DESIGN.md`；移动端以现有 APP 截图和用户参考原型为视觉标准。
- 首页：项目装备、甲方装备双页签；任务采用卡片列表；右下角固定“新增盘点”。
- 录入页：直接从新增进入，不设置开始步骤；装备使用紧凑卡片展示基准数量、盘点数量和差异。
- 结果页：提交后进入只读结果，汇总正常、盘盈、盘亏数量，并逐项显示结果。
- 交互：数量支持步进录入；一物一码装备保留扫码入口的演示表达。
