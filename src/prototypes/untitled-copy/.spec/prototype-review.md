## 总体点评

列表业务主线已能表达装备从采购需求、采购计划、京东订单导入、入库、库存、调拨、盘点、出库到成本归集的生命周期。原型原先已经有总览流程卡片和各模块列表，但多数行内操作是统一的“编辑/删除”，不利于用户理解下一步业务动作。

本次已将关键列表操作改为按状态驱动的业务动作：采购需求可进入计划，采购计划可进入订单导入，订单可进入入库，入库可查看库存，调拨、盘点、出库可追到成本归集。整体串联能力已经从“页面并列展示”提升为“可按业务顺序走一遍”。

## P0-P3 优先级问题

### P2 - 部分关键动作仍是原型级表达，缺少提交后的状态反馈

- 证据：列表操作已补充“汇总生成计划、导入订单、创建入库、录码入库、确认出库、处理差异、查看成本”等动作，但多数按钮目前用于跳转或静态展示，未模拟保存成功、失败、部分成功或状态变更。
- 影响：评审时能理解业务方向，但还不能验证“点击后单据状态如何变化、库存和成本如何联动更新”。
- 修复方向：下一轮可为关键动作增加轻量反馈，例如成功提示、状态预览、下一步引导，重点覆盖订单导入、入库确认、出库确认和盘点差异处理。

### P2 - 顶部主操作与列表行操作还没有完全统一

- 证据：页面头部仍保留“新增库存调整、新增入库单、新增出库单”等通用主按钮，部分按钮未绑定弹窗或下一步页面；列表行操作已经更贴近业务状态。
- 影响：用户从列表行进入主流程比较顺，但从页面顶部发起业务时仍可能出现入口不一致。
- 修复方向：将页面顶部主按钮与对应列表业务动作统一：新增仓库、生成采购计划、选择订单文件、创建盘点任务、历史装备补录等都应有明确弹窗或引导。

### P3 - 查看明细和来源追溯可以继续补强

- 证据：采购计划、订单、入库、调拨、领用和成本归集列表已有来源单号和摘要，但“查看明细 / 查看来源”尚未展开单头多明细、价格口径、成本来源链路。
- 影响：当前能串起主流程，但对价格差异、单件成本、批次成本和项目成本的追溯还偏摘要。
- 修复方向：补一个通用单据详情抽屉，展示明细、来源单据、状态时间线、价格口径和成本归集结果。

## 完整性与项目对齐

原型范围与需求快照一致：覆盖装备档案、仓库、采购、订单导入、入库、库存、一物一码、调拨、盘点、出库和成本归集。京东慧采按方案五处理，当前只表达 Excel 订单导入，没有扩展到实时接口、物流、发票、对账和付款，符合一期范围。

列表操作已按业务链路补齐主要承接关系：需求到计划，计划到订单，订单到入库，入库到库存，调拨/出库/盘点到成本归集。删除限制也基本符合“已发生业务或已归集成本的数据删除受限”的决策。

## 业务逻辑连贯性

采购侧：需求列表按“待汇总 / 单独采购”区分处理，计划列表按“待采购 / 待导入订单 / 订单已导入 / 入库中”承接订单导入，订单列表按“待商品匹配 / 待入库 / 部分入库”承接入库。

仓储侧：入库列表展示录码进度，库存列表区分台账和一物一码装备，调拨列表体现待出库、在途、已完成，出库列表体现待出库和已出库。成本侧通过调拨、出库、项目专属采购等来源归集到项目。

## 状态、异常、边界与恢复

已检查状态覆盖：正常、低库存、待汇总、待采购、待导入订单、待商品匹配、待入库、部分入库、待出库、在途、差异待处理、盘盈审核中、已完成等。当前主要缺口是异常与恢复：导入失败、商品匹配失败、录码不足、库存不足、重复出库、盘点审核驳回等尚未展开。
