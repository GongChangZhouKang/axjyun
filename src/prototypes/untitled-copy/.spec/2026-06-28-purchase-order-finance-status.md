# 采购订单付款与发票登记需求设计快照

日期：2026-06-28

## 产品需求

- 目标用户：采购与财务经办人员。
- 核心任务：采购订单创建后，按实际付款和开票进度分次登记，在订单列表查看当前状态。
- 范围：新增付款状态、发票状态、付款登记、发票登记、附件元信息和订单详情历史记录。
- 状态规则：累计金额为 0 时显示未付款/未开票；大于 0 但小于订单金额时显示部分付款/部分开票；达到订单金额时显示已付款/已开票。
- 数据来源：复用当前采购订单示例数据，使用前端会话状态模拟新增记录。
- 不做：不接后端，不做权限、审批、记录修改、删除或冲销。

## 设计方案

- 设计基底：沿用当前采购页的蓝灰色后台风格、表格、状态标签和大弹窗，不切换主题。
- 信息架构：状态列位于订单金额与入库单状态之间；操作列保留两个独立的“登记付款”和“登记发票”入口。
- 交互路径：订单列表进入登记弹窗，查看订单总额、已登记金额和剩余金额，补充本次业务信息并提交，返回列表即时更新状态。
- 附件：单条记录支持一个 PDF/PNG/JPG 文件，上限 10MB；原型仅保留文件名、类型和大小。
- 验证：本次金额必须大于 0 且不得超过剩余金额；日期与流水/凭证号或发票号码必填；达到全额后禁用对应登记操作。

## 决策记录

- 选择支持分次付款和分批开票，而不是手工更新状态。
- 选择标准业务字段并增加附件上传。
- 选择两个独立行内操作，不合并为统一财务登记入口。
- 选择在查看订单中展示累计金额和完整历史记录。
