# 装备档案管理 7 月迭代决策快照

## 产品需求
- 目标用户：安保装备管理员。
- 核心任务：维护装备档案、多级装备分类和分类编码。
- 页面范围：装备档案、分类管理两个标签页；不扩展库存、采购、审批等其他模块。
- 数据来源：现有系统截图与 `untitled-copy` 装备档案页中的分类数据。
- 功能：分类多级维护、分类编码、装备编码列、供应商联想、多级分类选择、编码自动生成。
- 验收：页面接近现有系统，关键控件可操作，新增/编辑/筛选/分页/状态切换与校验可演示。

## 设计方案
- 设计基底：`src/themes/contract-detail/DESIGN.md`。
- 信息架构：页面顶部为“装备档案 / 分类管理”标签；各标签内采用筛选区、操作区和数据表格。
- 分类管理：树形表格展示分类层级、分类编码、装备数量和状态。
- 装备弹窗：Element 表单；供应商联想输入；分类级联选择；装备编码只读预览。
- 编码规则：按末级分类编码追加三位流水号，例如 `FZ-YD-001`。
- 视觉方向：高密度中文政企后台，白色与浅灰表面、克制边框、蓝色主操作、4px 圆角。
- 技术边界：Axhub React 入口作为挂载壳，页面主体由 Vue 3 + Element Plus 实现。

## 实施边界
