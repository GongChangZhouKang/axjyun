商品价格有波动时，安保管理平台里**成本归集不能简单按“当前商品价格”计算**，而应按业务节点固化价格，并区分“预算成本、采购成本、入库成本、领用成本、项目成本、结算成本”。

核心原则是：

> **预算看预算价格，采购看订单价格，库存看入库价格，项目成本看领用价格，财务看发票价格。**
>

---

## 一、价格波动下的成本归集口径
建议把价格拆成 5 类。

| 价格类型 | 取价时点 | 用途 |
| --- | --- | --- |
| 预算价格 | 采购申请时 | 做预算校验、审批依据 |
| 订单价格 | 京东慧采下单时 | 形成采购订单金额 |
| 入库价格 | 收货入库时 | 形成库存成本 |
| 领用价格 | 项目/人员领用时 | 归集项目成本 |
| 发票价格 | 发票/结算时 | 财务最终核算 |


这样即使商品价格每天变化，也不会影响已经发生的历史业务。

---

# 二、推荐成本归集逻辑
## 1. 采购申请阶段：锁定“预算价”
项目或驻点发起采购申请时，系统从京东慧采获取当时的商品价格，生成申请单。

```latex
申请数量 × 申请时价格 = 预算占用金额
```

例如：

```latex
反光背心 100 件
申请时单价：28 元
预算占用：2,800 元
```

这个价格用于审批和预算冻结，但**不作为最终成本**。

---

## 2. 下单阶段：锁定“订单价格”
审批通过后，如果京东慧采价格发生变化，系统应重新校验价格。

例如预算价格为 28 元，订单价格变成 30 元。

此时有两种处理方式：

### 方式 A：价格变化在容差内，自动下单
比如设置价格浮动阈值为 ±5%。

```latex
预算价格：28 元
订单价格：30 元
涨幅：7.14%
```

如果超过阈值，则进入重新审批；如果没有超过，则自动下单。

### 方式 B：价格超过阈值，重新审批
适合金额较大或预算严格的企业。

```latex
价格上涨超过 5%
系统触发：重新提交审批 / 采购确认
```

建议安保平台支持这个规则：

```latex
订单价格 ≤ 预算价格 × 价格浮动阈值
```

例如：

```latex
允许浮动 5%
预算价格 28 元
最高允许订单价格 29.4 元
实际订单价格 30 元
需要重新审批
```

---

## 3. 收货入库阶段：形成“库存成本”
商品签收后，平台根据实际采购订单金额入库。

```latex
入库成本 = 实际下单金额 + 运费 + 税费 + 其他采购附加成本
```

如果京东包邮、含税价采购，可以简化为：

```latex
入库成本 = 实际下单含税金额
```

这一步形成库存台账。

```latex
商品：反光背心
数量：100 件
入库单价：30 元
库存金额：3,000 元
```

---

## 4. 领用阶段：归集“项目成本”
项目成本不建议在采购申请时就直接确认，应该在**领用、调拨、消耗**时确认。

例如总部统一采购 100 件反光背心，后来分配给不同项目：

| 项目 | 领用数量 | 单价 | 项目成本 |
| --- | ---: | ---: | ---: |
| A 小区项目 | 30 件 | 30 元 | 900 元 |
| B 工厂项目 | 50 件 | 30 元 | 1,500 元 |
| C 商场项目 | 20 件 | 30 元 | 600 元 |


这样项目成本才准确。

---

# 三、库存成本计算方法
安保用品和装备可以用不同计价方式。

## 方案一：移动加权平均法，推荐用于耗材
适合：

保安服、手套、反光背心、雨衣、电池、办公耗材等。

公式：

```latex
移动加权单价 = 当前库存总金额 ÷ 当前库存数量
```

例如：

第一次入库：

```latex
100 件 × 30 元 = 3,000 元
```

第二次价格上涨：

```latex
100 件 × 34 元 = 3,400 元
```

当前库存：

```latex
数量：200 件
金额：6,400 元
加权单价：32 元
```

后续项目领用时：

```latex
领用 10 件 × 32 元 = 项目成本 320 元
```

**优点：** 简单、稳定，适合价格经常波动的通用物资。

---

## 方案二：批次成本法，推荐用于装备/资产
适合：

对讲机、执法记录仪、巡更棒、防刺服、头盔、金属探测器等。

每一批采购保持独立成本。

例如：

| 批次 | 入库时间 | 数量 | 单价 |
| --- | --- | ---: | ---: |
| 批次 A | 6 月 | 20 台 | 380 元 |
| 批次 B | 7 月 | 20 台 | 420 元 |


如果项目领用的是批次 B，则项目成本按 420 元归集。

**优点：** 成本准确，适合资产追踪、维保、报废、折旧。

---

## 方案三：先进先出法，适合有保质期或批次管理物资
适合：

口罩、消毒用品、部分劳保用品、应急物资。

先入库的批次先被领用。

```latex
先消耗旧批次，再消耗新批次
```

---

# 四、安保管理平台建议采用的成本规则
我建议这样设计：

| 物资类型 | 推荐计价方式 | 成本归集时点 |
| --- | --- | --- |
| 一般耗材 | 移动加权平均 | 领用/消耗时 |
| 服装类 | 移动加权平均或尺码批次 | 发放给人员/项目时 |
| 高价值装备 | 批次成本法 | 领用或转资产时 |
| 固定资产 | 单件成本法 | 入账、折旧、调拨、报废 |
| 应急物资 | 先进先出法 | 出库时 |
| 办公用品 | 移动加权平均 | 领用时 |

# 六、价格差异怎么处理
价格波动会带来几类差异。

## 1. 预算价格与订单价格差异
用于审批控制。

```latex
价差 = 下单金额 - 申请金额
```

处理方式：

```latex
超过阈值 → 重新审批
未超过阈值 → 自动通过
```

---

## 2. 下单金额与发票金额差异
用于财务对账。

```latex
发票金额 ≠ 订单金额
```

常见原因：

优惠、运费、拆单、退货、税率变化、部分开票。

处理方式：

```latex
进入对账异常池
由采购/财务确认
```

---

## 3. 入库金额与项目领用金额差异
如果用移动加权平均，项目领用单价可能与某次采购单价不同，这是正常的。

系统应记录：

```latex
入库批次价
库存加权价
领用成本价
项目成本金额
```

---

# 七、推荐业务闭环
```latex
京东慧采实时价格
        ↓
采购申请时生成价格快照
        ↓
预算占用 / 审批
        ↓
下单前再次询价
        ↓
价格变化校验
        ↓
通过则下单，超阈值则重新审批
        ↓
按实际订单价格入库
        ↓
按库存计价规则出库
        ↓
领用到项目 / 人员 / 成本中心
        ↓
形成项目成本
        ↓
发票与订单对账
        ↓
价差、退货、售后调整成本
```

---

# 八、产品设计建议
在安保管理平台里，可以设计一个**价格与成本控制规则中心**。

主要配置项包括：

```latex
是否启用价格快照
价格有效期
价格浮动阈值
超阈值处理方式
库存计价方式
是否含税计成本
是否按项目归集
是否按人员归集
是否按资产编号归集
是否启用发票对账
```

# 九、建议你在方案里这样表述
可以写成：

> 针对京东慧采商品价格动态变化的特点，平台采用“价格快照 + 下单复核 + 成本分层归集”的机制。采购申请阶段锁定预算价格，下单阶段按京东实时价格复核并生成订单价格，收货入库后形成库存成本，项目领用或装备发放时按移动加权、批次成本或单件成本归集至项目、人员或资产台账。对于预算价格、订单价格、发票价格之间的差异，系统提供价格浮动阈值、重新审批、对账异常处理和成本调整机制，保障预算可控、成本真实、过程可追溯。
>

---

## 最推荐的成本归集方案
对于安保管理平台，我建议：

```latex
申请阶段：记录价格快照，用于预算审批
下单阶段：锁定订单价格，用于采购订单
入库阶段：按入库价格形成库存成本
出库阶段：
  - 耗材：移动加权平均
  - 装备：批次成本 / 单件成本
项目成本：在领用或消耗时归集
财务结算：以发票和对账结果做最终调整
```

这样既能应对京东慧采价格波动，也能保证安保项目成本核算准确。
